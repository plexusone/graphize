## [v0.2.0] - 2026-04-19

### Highlights

- Multi-language extraction: Go, Java, TypeScript, Swift
- External provider interface for adding language support
- Watch mode for auto-rebuild on file changes
- Git hooks for automatic analysis on commit/checkout
- Neo4j Cypher and Obsidian vault export formats
- Node explanation with community and centrality context
- Platform installers for Claude, Cursor, Copilot, Codex, Gemini, Aider

### Added

- `provider` package with `LanguageExtractor` interface for external language support ([`aea79bf`](https://github.com/plexusone/graphize/commit/aea79bf))
- Priority-based provider registry (Default, Thick, Custom levels) ([`aea79bf`](https://github.com/plexusone/graphize/commit/aea79bf))
- Go language extractor using native `go/ast` parser ([`7aa3e69`](https://github.com/plexusone/graphize/commit/7aa3e69))
- Java language extractor with Spring framework detection ([`7aa3e69`](https://github.com/plexusone/graphize/commit/7aa3e69))
- TypeScript/JavaScript extractor using tree-sitter ([`7aa3e69`](https://github.com/plexusone/graphize/commit/7aa3e69))
- Swift extractor with protocol and extension support ([`7aa3e69`](https://github.com/plexusone/graphize/commit/7aa3e69))
- `WithCustomExtractor` option for direct extractor injection ([`12d8c52`](https://github.com/plexusone/graphize/commit/12d8c52))
- `graphize path <from> <to>` command to find shortest path between nodes using BFS
- `graphize benchmark` command showing token reduction statistics (raw vs TOON)
- `graphize watch` command for auto-rebuild on file changes with debouncing
- `graphize hook install/uninstall/status` commands for git hook management
- `graphize export cypher` command for Neo4j CREATE statements
- `graphize export obsidian` command for wiki-style vault with wikilinks
- `--directed` flag for `graphize analyze` to control edge directionality
- `--html` and `--report` flags for watch mode to auto-regenerate outputs
- `--debounce` flag for watch mode to configure delay (default 500ms)
- post-commit hook: auto-run `graphize analyze` after commits
- post-checkout hook: check if graph is stale after checkout
- `graphize explain <node>` command with community membership, centrality scores, and neighbor details
- `graphize install <platform>` command for 6 AI assistant integrations (claude, cursor, copilot, codex, gemini, aider)
- `graphize report --health` flag for corpus health assessment with actionable verdicts
- `graphize enhance --include-docs` and `--docs-only` flags for markdown/text extraction
- Markdown extractor in `pkg/extract/markdown/` for documentation-to-code linking
- Platform installers in `pkg/platform/` for Claude Desktop, Cursor, Copilot, Codex, Gemini CLI, and Aider
- Corpus health check in `pkg/analyze/health.go` with token reduction metrics
- Node explanation in `pkg/query/explain.go` with community and centrality context

### Changed

- `extract` package now uses `provider` types with backward compatibility aliases ([`12d8c52`](https://github.com/plexusone/graphize/commit/12d8c52))
- CLI commands use `NewMultiExtractor(DefaultRegistry)` pattern ([`38f92f6`](https://github.com/plexusone/graphize/commit/38f92f6))
- Composite surprise scoring weights cross-file edges higher (2.5x) and adds code-doc bonus (+1.5)

### Documentation

- CLI reference updated with explain and install commands
- Getting started guide with watch mode and git hooks
- Export formats table updated with Cypher and Obsidian
- Phase 6 implementation archived in docs/plans/v0.2.0-plan.md

## [v0.1.0] - 2026-04-09

### Highlights

- LLM-powered tool to turn Go codebases into queryable knowledge graphs
- Two-step extraction: deterministic AST + optional LLM semantic analysis
- TOON export format with 98% token reduction vs JSON for AI agents
- MCP server for AI agent integration with 5 graph query tools

### Added

- `graphize init` command to create graph database directory structure
- `graphize add <repo>` command to track repositories with git commit hashes
- `graphize status` command to show sources with staleness detection
- `graphize analyze` command to extract AST graph from Go sources
- `graphize query` command with BFS/DFS traversal, depth control, and edge filters
- `graphize export html` command with Cytoscape.js interactive visualization
- `graphize export json` command in Cytoscape JSON format
- `graphize export toon` command for token-efficient agent output
- `graphize summary` command for Markdown summary in AGENTS folder
- `graphize report` command with god nodes, communities, surprises analysis
- `graphize enhance` command to prepare files for LLM semantic extraction
- `graphize merge` command to merge LLM-extracted semantic edges into graph
- Go AST extraction: packages, files, functions, methods, types, imports, calls
- SHA256-based per-file caching in `pkg/cache/cache.go`
- `--no-cache` flag for analyze command to bypass cache
- Cache hit/miss statistics reporting
- God nodes analysis: most connected entities excluding file-level hubs
- Community detection: package-based grouping with cohesion scores
- Surprising connections: cross-file, cross-community edge detection
- Isolated nodes detection for potential documentation gaps
- Package statistics: files, functions, types, imports per package
- Edge confidence breakdown: EXTRACTED vs INFERRED vs AMBIGUOUS
- Semantic edge types: inferred_depends, rationale_for, similar_to, implements_pattern, shared_concern
- `pkg/extract/merge.go` for parsing and validating LLM semantic JSON output
- `skills/enhance.md` Claude Code skill for semantic extraction workflow
- `agents/specs/semantic-extractor.yaml` multi-agent-spec definition
- `graphize serve` MCP server using official Go SDK (github.com/modelcontextprotocol/go-sdk)
- MCP tool: `query_graph` for BFS/DFS graph traversal from search terms
- MCP tool: `get_node` for node details by ID or label
- MCP tool: `get_neighbors` for adjacent nodes with edge info
- MCP tool: `get_community` for listing community members
- MCP tool: `graph_summary` for stats, god nodes, and suggested questions
- Suggested questions generation: ambiguous edges, bridges, inferred verification, isolated nodes, low cohesion
