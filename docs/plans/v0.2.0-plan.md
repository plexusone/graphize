# v0.2.0 Implementation Plan

This document archives the implementation plan for v0.2.0 features.

## Phase 5: Quick Wins (Complete)

All Phase 5 features have been implemented:

- `graphize path "A" "B"` - Trace exact path between nodes using BFS
- `graphize benchmark` - Print token reduction stats
- `--directed` flag for `graphize analyze`
- `graphize hook install/uninstall/status` - Git hooks for auto-analysis
- `graphize watch` - Auto-rebuild on file changes
- `graphize export obsidian` - Wiki-style vault export
- `graphize export cypher` - Neo4j Cypher statements

### 6.4 Explain Command

```bash
graphize explain "func_main"
# Output:
# Node: func_main (function)
# Community: 3 (cli commands)
# In-degree: 2, Out-degree: 15
# Neighbors: func_init, func_run, pkg_cobra...
# Called by: main
# Calls: runAnalyze, runExport, runQuery...
```

**Status:** Complete in `pkg/query/explain.go` and `cmd/graphize/cmd/explain.go`

### 6.5 Markdown/Text Extraction

Extract documentation concepts for linking to code entities:

```go
// pkg/extract/markdown/extractor.go
type Extractor struct{}

func (e *Extractor) Language() string { return "markdown" }
func (e *Extractor) Extensions() []string { return []string{".md", ".markdown", ".txt", ".rst"} }
func (e *Extractor) ExtractFile(path, baseDir string) ([]*graph.Node, []*graph.Edge, error)
```

Edge types:

- `documents` - Doc concept documents code entity
- `describes` - Doc describes behavior/architecture
- `explains` - Doc provides design rationale

**Status:** Complete in `pkg/extract/markdown/extractor.go`

**Integration:**

- `graphize enhance --include-docs` - Include doc files
- `graphize enhance --docs-only` - Only analyze doc files
